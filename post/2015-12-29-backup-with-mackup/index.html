Mac OS X Backup System

Dec 29, 2015

I recently expanded my collection of Mac OS X devices to include a new desktop system and immediately became irritated by having to re-install and re-configure all of my software. I realise this is a common problem and decided to investigate further. I came across an article which basically described a near perfect solution to this problem. This blog post is more of a personal reminder of how to backup my Mac OS X devices.

Configuration

Configuration of Mackup is incredibly simple (which is one of the main reasons I like it) as all you need to do is provide a valid configuration file and then execute one simple command. Your configuration file can be empty if you wish to use the ~/Dropbox/Mackup directory for you backups. If, like me, you prefer to avoid putting all your data on a remote server that you don’t own then you can tweak the default configuration file ~/.mackup.cfg to use a local folder. My particular ~/.mackup.cfg file is as follows:

[storage]
engine = file_system
path = Documents/Computing

Execution

Once you have your configuration in place the following two commands are fairly self explanatory:

$ mackup backup
$ mackup restore
